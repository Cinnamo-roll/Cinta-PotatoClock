# Repository guidance

## Attribution and licensing

- Preserve the repository's LICENSE, NOTICE, author metadata, original repository URL, and existing copyright notices.
- Do not remove or rewrite attribution unless the repository owner explicitly requests it.
- Do not claim third-party or contributor-owned code as belonging solely to the original author.
- When adding new source files owned by this project, follow the repository's established copyright-header convention where appropriate.
- Keep product attribution unobtrusive and place it in About, documentation, package metadata, or version information rather than the primary user workflow.
- Never add telemetry, tracking, fingerprinting, hidden watermarks, license enforcement, remote kill switches, or obfuscated attribution mechanisms.
- Preserve third-party licenses and notices.
- Before changing the project license, report compatibility and ownership concerns instead of changing it automatically.
